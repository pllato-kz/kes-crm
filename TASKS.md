## Интеграция с 1С (OData)

Стандартный REST/OData-интерфейс 1С:Предприятие (`/<база>/odata/standard.odata/`). Цель — не дублировать данные руками, а тянуть из учётной системы.

- 🟥 P1 — Подключение к OData 1С: настройки базы (URL, логин/пароль или токен), хранить на стороне Worker, не в браузере. Включить публикацию OData в 1С.
- 🟥 P1 — Импорт номенклатуры: `Catalog_Номенклатура` → `products` (SKU, наименование, единица, категория). Связать с импортом прайса EKF.
- 🟥 P1 — Импорт контрагентов: `Catalog_Контрагенты` → `clients` (БИН, наименование, тип). Дедупликация по БИН.
- 🟥 P1 — Остатки склада: регистр `AccumulationRegister_ТоварыНаСкладах/Balance` → `stock`/`reserved`. Периодическая синхронизация.
- 🟥 P2 — Выгрузка сделки в 1С: создание `Document_СчетНаОплатуПокупателю` / `Document_РеализацияТоваровУслуг` из сделки на этапе «Счёт выставлен»/«Отгружено».
- 🟥 P2 — Сверка взаиморасчётов: тянуть баланс клиента (`balance`) из 1С вместо ручного.
- 🟥 P2 — Маппинг справочников: таблица соответствия id KES ↔ `Ref_Key` (GUID) 1С, чтобы синхронизация была идемпотентной.
- 🟥 P3 — Инкрементальная синхронизация по дате изменения, обработка `$filter`/`$top`/`$select`, пагинация.
- 🟥 P3 — Логи и статус синка в «Настройках»: когда последний раз тянули, сколько записей, ошибки.

> Заметка: OData-вызовы идут через Worker (CORS + хранение кредов), браузер напрямую в 1С не ходит. Связано с задачей «Поднять Cloudflare Worker + D1».
