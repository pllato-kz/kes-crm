# SIP-софтфон (WebRTC) — что встроено в KES CRM и как включить

Модуль «звонки прямо из браузера» подключён в проект. Голос идёт **браузер → ваш
Asterisk → SIP-trunk Binotel → телефон клиента**. Бэкенд в аудио не участвует —
только выдаёт SIP-креды и пишет лог звонка.

## Что уже сделано в коде (готово, задеплоено)

**Бэкенд** (`functions/api/[[path]].js`):
- `GET /api/sip/token` — выдаёт браузеру SIP-креды из секретов окружения. Если
  секреты не заданы → `503 sip_not_configured` (фронт тихо прячет UI).
- `POST /api/sip/log` — пишет звонок в историю (таблица `messages`, `channel='call'`):
  направление, длительность, привязка к сделке/клиенту (клиент ищется по номеру).

**Фронт**:
- `sip-client.js` — браузерный софтфон (SIP.js 0.21.2 с CDN, ленивая загрузка),
  адаптирован под этот проект (токен `kes_jwt`, тот же origin, `/api/sip/*`).
  Рисует свой UI: нижний бар статуса + оверлей активного звонка + DTMF.
- Подключён в `index.html`; пре-варм после входа в `bootApp()` (UA регистрируется
  заранее → первый звонок мгновенный). Если SIP не настроен — ничего не показывает.
- Стили — секция «SIP CLIENT» в `styles.css`.

**Кнопка 📞** в карточках — оставлена на текущем **Binotel-callback** (по вашему
решению). Софтфон сейчас: регистрируется, принимает **входящие** в браузере и
логирует звонки. Исходящий через софтфон — `window.SipClient.call(phone, {dealId, clientId, contactName})`
(можно повесить на отдельную кнопку или переключить 📞 одной строкой, когда захотите).

## Что нужно сделать вам, чтобы звонки заработали (Фаза 1 — инфраструктура)

Это разовая настройка вне проекта (детали — в `README.md` и
`asterisk-setup-README.md` исходного модуля):

1. **VM с публичным IP**, Ubuntu 22.04 (подойдёт **GCP e2-micro Always Free**, $0).
   - Зарезервировать статический External IP.
   - Открыть порты: TCP `22,80,443,8089`; UDP `5060,10000-20000,3478`.
2. **Binotel**: открыть тикет в support — добавить IP вашей VM в whitelist для
   SIP-аккаунта; получить SIP-trunk (логин/пароль/хост, обычно `sipNN.binotel.com`),
   нужен в обе стороны (исходящие + входящие).
3. **Asterisk**: на VM запустить `setup-asterisk.sh` (из исходного модуля) с данными
   Binotel-trunk. Скрипт поставит Asterisk 22 + TLS (WSS) + DTLS + endpoint `100` +
   trunk + диалплан. В конце распечатает `SIP_DOMAIN` (вида `<ip-через-дефис>.nip.io`)
   и `SIP_ENDPOINT_PASSWORD`.

> ⚠️ Не Oracle Cloud Free (1:1 NAT ломает WebRTC). Нужен прямой public IP (GCP).

## Активация в CRM (Фаза 4 — секреты)

Задать секреты Cloudflare Pages (Settings → Environment variables / `wrangler pages secret put`):

| Секрет | Обяз. | Пример | Что |
|---|---|---|---|
| `SIP_DOMAIN` | да | `34-23-1-1.nip.io` | домен Asterisk |
| `SIP_ENDPOINT_PASSWORD` | да | hex32 | пароль pjsip-endpoint `100` (из setup-asterisk.sh) |
| `SIP_USER` | нет | `100` | SIP-username (по умолчанию `100`) |
| `SIP_TURN_URL` / `SIP_TURN_USERNAME` / `SIP_TURN_PASSWORD` | нет | — | TURN, если оператор за жёстким NAT |

После задания секретов и редеплоя:
1. Открыть CRM по HTTPS (хард-reload) → внизу справа появится бар «✓ Готов к звонкам».
2. В консоли проверить: `await SipClient.call('9000')` (эхо-тест), затем реальный номер.
3. Входящие пойдут на endpoint `100` (зазвонит в браузере).

## Включить исходящий через софтфон на кнопке 📞 (когда решите)

В `app.js`, функция `callPhone(phone, ctx)` сейчас зовёт `binotel/call`. Чтобы
звонить через браузерный софтфон, заменить тело на:
```js
if (window.SipClient && window.SipClient.isReady) {
  return window.SipClient.call(phone, { dealId: ctx && ctx.dealId, customerId: ctx && ctx.clientId });
}
// иначе — текущий Binotel-callback (fallback)
```

## Безопасность
- `sip-client.js` отдаётся как статика (нужен браузеру) — в нём нет секретов.
- SIP-пароль выдаётся только авторизованному пользователю через `/api/sip/token`.
- `*.md` (включая эти заметки) не публикуются (`.assetsignore`).
