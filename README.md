# KES CRM — мокап

Интерактивный фронт-прототип CRM для **ТОО KazEnergoSnab** — карагандинского дистрибьютора электротехники EKF.

## 🌍 Демо

**https://kes-crm-2vx.pages.dev** — заходи и кликай.

Логин-экран показывает 4 демо-роли (Директор / Менеджер / Кладовщик / Бухгалтер) — клик по плитке → внутрь. Каждая роль видит свой набор разделов.

## Стек

- Vanilla **HTML / CSS / JS** (ES modules не используется, всё в `<script src=>`)
- Демо-данные в `data.js`, состояние в `localStorage`
- Графики на [Chart.js](https://www.chartjs.org/) (через CDN)
- Деплой: **Cloudflare Pages**

## Структура

```
index.html    — оболочка, подключение скриптов, лоадер
styles.css    — дизайн-система (брендинг KES: #00A6E2 + чёрный)
data.js       — демо-данные (SEED), роли, права, session/state helpers
app.js        — рендер 12 модулей, роутер, формы, печать СФ
```

## Что есть

- 4 роли: Директор, Менеджер по продажам, Кладовщик, Бухгалтер — у каждой свой набор модулей
- 12 модулей: Дашборд, Заявки, Сделки (Kanban + drag-drop), Клиенты, Каталог (21 категория, ~25 SKU EKF), Склад, Отгрузки, Документы, Поставщики, Задачи, Отчёты (Chart.js), Настройки
- Реальные позиции в сделках (товары из каталога, авторасчёт суммы)
- Печать счёта-фактуры с реквизитами KES и суммой прописью
- Админка пользователей (только директору) + матрица прав
- Всё хранится в localStorage; сброс — кнопка в сайдбаре

## Локальный запуск

```bash
python3 -m http.server 8805
# открыть http://127.0.0.1:8805
```

## Деплой

CF Pages подключён к этому репо — `git push` в `main` автоматически деплоит на `kes-crm-2vx.pages.dev`.

Ручной деплой через wrangler (если нужно):
```bash
wrangler pages deploy . --project-name=kes-crm-2vx
```
